Resumo da Disciplina

A disciplina tem como objetivo apresentar os princípios fundamentais da Engenharia de Software, enfocando a construção sistemática, disciplinada e mensurável de sistemas. O conteúdo abrange os principais modelos de processo de software, com ênfase no RUP (Rational Unified Process), além de técnicas para levantamento, modelagem e documentação de requisitos utilizando a linguagem UML. A proposta é preparar o aluno para projetar soluções robustas e alinhadas com os padrões adotados pela indústria de desenvolvimento de software.

Conteúdos Desenvolvidos

  • Fundamentos e objetivos da Engenharia de Software
  • Modelos de processo de software
  • Processo Unificado (RUP): fases, atividades e papéis
  • Engenharia de requisitos: identificação, elicitação e especificação
  • Requisitos funcionais vs. não funcionais
  • Modelagem de requisitos com UML:
    • Atores
    • Casos de uso
  • Modelagem de sistemas com UML:
    • Diagramas de casos de uso
    • Diagramas de classes
  • Projeto de arquitetura de software
  • Padrões arquiteturais: camadas, componentes e responsabilidades

Exemplo de especificação de caso de uso:

Caso de Uso: Realizar Login
Atores: Usuário
Pré-condições: O usuário deve estar cadastrado no sistema
Fluxo Principal:
1. O usuário informa email e senha
2. O sistema valida as credenciais
3. O sistema redireciona para a página inicial
Fluxos Alternativos:
3a. Credenciais inválidas: sistema exibe mensagem de erro

Abordagem Metodológica

A disciplina é desenvolvida por meio de uma combinação entre teoria e prática, promovendo a aplicação dos conceitos em cenários reais de desenvolvimento de software. As aulas incluem:

  • Exposições teóricas com apoio de slides e quadro
  • Análise de estudos de caso reais para exercitar requisitos e modelagem
  • Atividades práticas utilizando ferramentas de modelagem UML
  • Projetos em grupo que simulam etapas completas do ciclo de desenvolvimento
  • Debates e discussões para consolidação de conceitos
  • Avaliações contínuas, com provas teóricas e entregas práticas

Competências Desenvolvidas

Ao final do curso, o aluno estará apto a:

  • Compreender os fundamentos da Engenharia de Software e sua relevância
  • Avaliar e aplicar diferentes modelos de processo de desenvolvimento, com foco no RUP
  • Levantar, organizar e documentar requisitos de maneira clara e precisa
  • Utilizar UML para modelar sistemas com atores, casos de uso e diagrama de classes
  • Estruturar a arquitetura de sistemas, identificando componentes, camadas e suas responsabilidades
  • Adotar práticas organizadas e orientadas à qualidade no desenvolvimento de software, favorecendo a manutenção, evolução e clareza técnica dos projetos

Exemplo de fases do RUP:

1. Concepção (Inception):
  - Definir escopo do projeto
  - Identificar stakeholders
  - Estimar custos e prazos

2. Elaboração (Elaboration):
  - Refinar requisitos
  - Desenvolver arquitetura base
  - Mitigar riscos principais

3. Construção (Construction):
  - Implementar funcionalidades
  - Desenvolver componentes
  - Preparar versão beta

4. Transição (Transition):
  - Implantar sistema
  - Treinar usuários
  - Realizar ajustes finais
