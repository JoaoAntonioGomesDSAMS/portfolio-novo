Introdução à Disciplina

A matéria de Projeto Integrador I tem como proposta principal colocar os alunos em uma experiência prática que simula o cotidiano de uma equipe de desenvolvimento. Durante o semestre, os estudantes mergulham em um projeto real, enfrentando todas as fases do processo de criação de software — desde a concepção da ideia até a entrega final.

Mais do que apenas aplicar conceitos técnicos, a disciplina busca desenvolver competências essenciais para a vida profissional, como trabalho em equipe, organização e resolução de problemas.

Controle de Versão com Git e GitHub

Dentro do escopo da disciplina, um dos aprendizados centrais é o controle de versão, algo fundamental para quem desenvolve software. Por meio do Git e do GitHub, aprendemos a gerenciar e colaborar em projetos de forma estruturada e segura.

Durante as aulas, exploramos temas como:

  • Introdução ao Git e GitHub: o que são e para que servem;
  • Diferenças entre o repositório local (Git) e o remoto (GitHub);
  • Inicialização e configuração de projetos com Git;
  • Criação, clonagem e organização de repositórios;
  • Uso de ramificações (branches) para organizar o fluxo de desenvolvimento;
  • Principais comandos:
    • git init — inicia um novo repositório;
    • git clone — copia um repositório remoto;
    • git add — prepara arquivos para commit;
    • git commit — registra as mudanças;
    • git push / git pull — envia/recebe alterações;
    • git branch / git merge — cria e integra ramificações;
  • Como lidar com conflitos de código;
  • Estratégias de colaboração com pull requests;
  • Boas práticas no uso do versionamento em equipe.

Projeto em Equipe: Do Zero ao Sistema

O grande desafio é tirar um projeto do papel e transformá-lo em um sistema funcional, trabalhando em equipe e aplicando os conhecimentos de forma integrada. Toda a jornada de desenvolvimento é vivida na prática, etapa por etapa.

As fases do projeto incluem:

  • Entendimento das necessidades (requisitos);
  • Escolha de tecnologias e avaliação da viabilidade do sistema;
  • Distribuição de funções e tarefas dentro da equipe;
  • Criação de protótipos para simular o funcionamento da interface;
  • Modelagem do banco de dados e definição das entidades;
  • Implementação do front-end, focada na experiência do usuário;
  • Codificação do back-end, com lógica, rotas e integração de dados;
  • Testes e validações, garantindo que o sistema funcione corretamente;
  • Elaboração da documentação técnica e do manual de uso;
  • Apresentação final, simulando a entrega a um cliente real.

Competências Aprofundadas

Mais do que conhecimento técnico, o Projeto Integrador também fortalece o desenvolvimento de habilidades interpessoais, valorizadas por empresas de tecnologia e inovação:

  • Colaboração ativa e sinergia entre colegas;
  • Organização e planejamento de tarefas;
  • Comunicação eficaz entre membros da equipe;
  • Autonomia, responsabilidade e iniciativa;
  • Capacidade de adaptação a novos desafios;
  • Pensamento analítico e foco em soluções;
  • Comprometimento com entregas e qualidade do projeto.
